Vue基础

1.什么是Vue？

Vue 是一个能为用户提供数据渲染的渐进式框架 有一定的规则和约定

2.Vue的使用

首先需要导入Vue包（开发模式和生成模式） 开发时用开发模式 更齐全

创建Vue实例绑定选择器 配置相关属性

Vue响应式特性： Vue框架中数据一旦改变 相应的视图也会更着改变

3.Vue中的指令

Vue的指令相当于标签的作用 前缀为v- 不同的指令实现不同的操作

v-html: 相当于innerHtml标签的作用 在一个页面插入html元素

v-show: 元素的显示和隐藏 语法： v-show=”表达式“ 注意表达式必须是一个结果 当值为true 表示显示 值为false表示隐藏

v-if ： 元素的条件显示和隐藏 语法： v-if=”表达式” 一般用于需要有条件的时候
v-else if : 辅助v-if 再次进行条件判断 语法：v-else-if = “表达式”
v-esle: 辅助v-if指令 必须和v-if出现 不能独自出现

v-show和v-if的不同之处：
v-show本质上是将元素的css样式进行改变 – display：none 进行隐藏 适用于频繁的显示隐藏
v-if： 本质上是需要条件熏染 当满足什么时候在进行显示或隐藏 每次显示或隐藏 底层是创建或移除 并不是修改css样式 适用与不平凡显示的情况

v-on: 用于注册事件 比如监听事件 比如点击事件 语法： v-on: 事件名=“表达式”
v-on: 可以简写为 @ 如： @click 表示注册了一个点击事件 @click = v-on:click

表达式的值可以调用vue实例methods中的函数： 可以携带参数 如： @click = fn(1,2)

v-bind: 用于标签属性的动态绑定 语法： v-bind:属性名=“值” 可以简写为： :属性名=“值” 如： :src= www.baidu.com
v-bind == :

v-for： 用于循环渲染数据 语法： v-for = “(item,index) in 数组或者列表对象名 :key= “值””
v-for会将数组或对象中的所有元素全部渲染出来 key用来绑定唯一标识 一般用于动态绑定 使用 :key
绑定了key 删除的时候可以将元素之间移除 而不是默认的底层替换 并没有移除

v-model: 用于表单元素， 实现表单数据的双向绑定 – 双向绑定是指当数据或者视图有一方改变另外一方也会改变
语法： v-model=”属性名”

4.Vue工程化

利用Vue进行前端时 一般会进行工程化开发 ： 生成Vue工程化结构

利用node.js里的npm软件包进行下载vue版本 在工程目录中使用 npm create 项目名字（不能出现中文）自动初始化生成项目
项目搭建之后 使用VsCode打开

1.Vue工程项目的结构

package.json – 整个项目的配置文件 可以记录项目的作者 名字 项目控制台命令 以及项目所需要的第三方依赖包 以及版本号等等

vue.config.js – vue配置文件 可以设置Vue组件的命名规范校验是否开启等等

node_modules – 项目所需要用到的第三方依赖包 全部会下载到这个文件夹中

public下的index.html – 项目生成html页面的模板 一般不用管

Vue.app – 根组件 结构包含 template、style、script 也就是html css js三部分内容 根组件是整个项目的父组件

main.js – 项目的入口文件 在这里面导入了vue的包使用 创建了Vue实例，将Vue.app根组件与index.html容器进行挂载 挂载成功之后 根组件的内容会自动映射到index.html 从而生成一个页面

2.Vue工程的组件化开发

在Vue项目中 一个页面是分为多个组件来进行独立开发 最后进行整合在一块 也是为了提高开发的效率 高内聚低耦合
Vue.app 作为整个项目的根组件 也是父组件 可以将其他组件再次页面进行展示

普通组件的注册： 一般在src项目文件夹下创建一个compones文件夹用来存放项目的各个组件模块
每一个组件 也就是.vue文件的结构都是template、style、script三种 可用< 快捷生成模板
组件的命名需要是大驼峰或者使用 - 分割开 Vue底层会对组件名称进行校验 如不满足这两种会进行报错 可以在vue.config.js中关闭这种校验
组件命名之后 生成对应的内容 在父组件中引入： 在script中 导入子组件的包 import 包名 from ‘包所在路径’ 包名最好采用规范命名
然后在export default { 注册所需要的组件： components: { 组件名： 组件对象 }} 组件名和组件对象一样时可以只写一个（一般采用这种）
注册时所用到的组件名也是需要遵循vue的命名规范

注册成功之后可以进行子组件的使用 默认使用方式是 利用标签符号使用 如： <注册的组件名> 这样即可成功使用子组件

在项目中的export default中的components注册的组件 是局部组件 也就是只能在当前范围组件使用 在其他组件内不可以使用

可以进行全局组件的注册 在main.js入口文件中 使用 Vue实例对象进行全局注册的注册 如： App.component(组件名) 需要提前导入组件对象

在export default中 相当于是Vue实例对象中提供的一些方法 如methods、钩子函数等等都可以使用
但是需要注意的是 data的使用是一个函数 不再是一个对象，这是因为data需要每次在组件实例创建时调用 生成一个新的对象 从而保证每个实例所拥有的数据是独立的

在style样式中 可以控制样式的范围 默认访问是全局范围– 注册的css样式在其他组件也能使用 当项目比较大这种不太友好 可以在style标签上加一个scoped属性，这个属性添加之后 会将里面的样式作用范围限制在当前组件中从而不影响其他组件
scoped属性底层原理是为当前的组件生成一个哈希值 在当前组件的style样式选择器上会自动添加这个哈希值 从而保证每个组件中哈希值不一样 所使用的样式也就作用不到其他组件

5.组件间的通信 – 父与子通信以及非父子通信

1.父与子通信：

在Vue组件中 如果子组件想要访问父组件的数据 需要父组件在使用子组件对象的时候加上 一个自定义动态属性将数据传递过来，子组件在export default中使用 props：对象进行接受 如： props: { 传递过来的属性名: 类型} – 接受使用的属性名必须与父组件自定义的动态属性名一致 类型表示限制的类型不如会报错 ， 子组件接受之后可以使用此数据 直接使用属性名就是当前传递的数据

子组件不能直接修改父组件传递的数据 因为数据只存在于父组件中 子组件需要传递的话得利用 $emit 发送事件给父组件进行操作 如: this.$emit(‘事件名’,参数)
父组件需要在使用子组件的标签属性中 注册一个监听事件 来接受子组件传递的消息 如： <Son @add=”add”> 需要注意的是接受注册的事件名必须和传递的事件名保持一致 然后在提供方法在父组件中进行修改数据即可

注意： props对象中可以对传递过来的数据进行多个操作 如校验长度等等 这里我们只做了最简单的校验 就是类型的校验

2.非父子通信

其他vue组件之间的通信为非父子通信 一般使用 事件总线进行通信 – 事件总线本质上就是一个空的vue实例 利用的是vue的事件特性，我们需要创建一个空的vue实例进行导出， 然后在组件中导入这个vue实例， 发送消息的时候使用 事件总线对象名.$emit(‘事件名’，数据)发送数据 也可以提供数据
在接收方使用 事件名.$on(‘事件名’，回调函数) 如： Bus.$emit(“msg”， 123) Bus.$on(‘msg’, (msg) => { })

需要注意的是 $emit是发送事件 $on是接受事件 发送事件的名字必须和接受事件的名字保持一致 才可以接受到

跨层级的通信 – 父组件与子孙组件的通信
当跨层级通信的时候 ，再使用父组件与子组件的通信非常麻烦 此时可以使用provide 和inject provide是一个对象 在需要传递的组件当中声明数据 如父组件
在子孙组件当中使用 inject注册当前的数据 即可直接

6.v-model详解：

本质上是一个动态绑定数据 + 接受修改之后的数据 :value= “msg” @input=”event.target.value” 一个是动态改变 一个是监听视图的改变
v-model底层直接帮我们封装好了一些表单数据的简化 不同的表单元素使用v-model绑定的底层实现不同danshi 思路一样

如果是父子组件中表单数据的修改可以使用这个来简化 父组件中提供数据过去 子组件中接受的属性名使用value: 发送的事件名为input 这两个必须固定
这样在父组件使用子组件标签属性中直接 使用v-model=”数据” 就可以完成表单数据的双向绑定 唯一的缺点是子组件必须使用value名来接受传递的数据
而传递的事件名也必须是input才可以完成这个简写

为了解决value属性名以及input事件名必须固定的写法 vue中提供了一种更好的机制也就是sync属性 –在动态传递的属性名加上sync后缀表示实现这个数据的双向绑定，但是必须在子组件发送事件名是update: +事件名， 接受的属性名可以自定义

ref获取Dom元素 我们使用普通选择器绑定的时候作用范围是整个页面 当我们像更加精确的绑定Dom元素的时候 我们需要使用到ref
给需要的Dom元素添加一个ref属性 =”ref属性名“ 获取的时候只需要$refs=” “ 就可以拿到当前的Dom元素从而实现操作
如： ref=”get” this.$refs=”get” 就可以获取到当前ref=get的这个Dom元素
可以使用ref来获取当前dom中的方法 – 拿到dom元素之后也可以拿到dom元素所包含的所有元素，以及里面封装的方法

Vue异步更新的特性：
Vue并不是实时进行更新的 ， 使用的是异步策略 这就会导致一个问题 无法及时拿到Dom 所以我们需要使用到$next.Tick( 回调函数)
next.Tick可以实现异步消息的接受 会等Dom元素渲染完成之后接受元素

7.自定义指令

directive封装 – 分为局部注册和全局注册指令

1.局部注册指令

在组建script中： 使用 directives：{ 指令名： { inserted(el,binding){}, update(el,binding){} }}
使用自定义指令直接在标签属性上加上 v-指令名

inserted（）函数 表示当前指令加入时所需要操作的业务逻辑
update（） 函数 表示指令绑定的dom元素发送变化需要改变的逻辑 一般是在数据变化时使用

el参数表示当前指令所作用的Dom元素 可以通过el对当前Dom元素进行操作
binding 参数可以用来拿到当前指令绑定的指令值 通过binding.value可以拿到当前指令对应的指令值

2.全局指令

在Vue实例进行注册
Vue.directive(‘ 指令名 ‘, { inserted(){}, update(){} }) 全局注册的指令在每个组件中都能使用 局部注册的指令只能在当前注册的组件中使用

自定义指令可以用来封装统一的操作 如：聚焦等

8.插槽的使用

slot标签为一个插槽 作用是用来占位 当使用插槽时 填入什么内容就会自定填充到插槽 可以对灵活的数据进行使用
slot可以设置一个默认值 ， 如： 使用插槽 slot文本的值就是默认值 ，当没有往插槽中填入数据时展示默认值

slot可以取一个name 用来区分不同插槽 使用时可以通过v-slot:插槽名 来进行锁定插槽范围 v-slot:可以简写为# #head head为插槽名字

作用域插槽： 插槽可以自定义添加属性=“值” 进行传递给使用插槽的部分 通过#插槽名 = 参数名来接受
传递的参数默认是一个对象 如： { data： ， msg： } 通过接受的参数名拿到这个对象即可进行操作

9.Vue路由

什么是路由？
路由是一种映射关系， 在Vue中，路由是路径和组件之间的映射关系

1.路由的使用

下载vue-router包
引入vue-router – import VueRouter from ‘vue-router’
全局注册路由 – Vue.use(VueRouter)
创建路由对象 – const router = new VueRouter({})
将路由对象和vue实例进行关联 router: 路由对象名字

最核心的配置： 创建路由与vue实例进行关联之后 可以在创建路由对象的时候配置路由规则， 然后使用router-view标签来使用当前路由
配置路由规则:
const router = new VueRouter( { routes: [ { name: FindMusic path: ‘/search’, component: Find} ]})
routes是一个数组形式 表示多个路由规则可以配置； name为组件的名字， path为路由路径 component是对应的组件
将path和component组件关联映射，当访问到这个路径的时候自动使用映射的Find组件（需要导入组件）

路由一般单独放在一个文件配置： router/index.js 配置路由 更好的进行管理

router-link标签的使用： 本质上是a标签的作用 区别是router-link在填写链接是使用的是to属性 而且不需要携带# 底层默认实现携带
带有router-link-active样式属性 和 router-link-exact-active样式属性 可以用来配置标签的样式自定义
语法： 发现

由于router-link标签默认带有样式属性，我们可以通过样式属性来设置css样式，也可以为样式属性更改一个名字
需要在路由配置中进行更改： 一个是exactclass 一个是activeclass 当设置好名字之后，可以用自定义的名字来访问样式属性从而设置相关值

2.路径参数传递和默认参数传递

在上述的router-link中的to路径里 我们可以携带参数过去 有两种方式 一种是默认传参 需要我们以键值对来进行传递
如： ‘/search?name=zhangsan&age=20’ 的形式传递过去
接受这个查询参数我们可以使用 $route.query.参数名来接受传递过来的参数

如果使用动态传递参数，需要我们在路由配置规则中的path路径里添加一个动态参数名来接受传递的参数
如： path: ‘/search/:words? ‘ :words表示words是一个动态参数名 接受传递的参数 分号：不可省略， ？表示传递的参数可选
动态传递的参数接受形式略有不同 使用 $route.params.动态参数名来接受传递的参数

查询参数传递和默认参数传递使用的场景分别是： 前者适用于多个参数传递， 后者适用于一个参数的传递

在创建路由对象的时候还可以指定路由重定向地址，写法： path: ‘/search’, component: Find, redirect: ‘/home’ 表示当匹配/search路径时跳转到home路径去
在最后一条配置我们可以配置一个当所以路径为被匹配到时返回一个错误的页面 如： path: ‘*’ ，component: Error 需要提供一个Error组件

每一个组件可以设置一个名字： name = “值” 如果未设置 则默认使用组件文件名作为组件名

路由跳转传参还有两个写法 ： this.$router.push( { path: query: { 参数名 }} ) 也可以不写路径写name 和参数 前提是要在路由规则中配置name组件名字
